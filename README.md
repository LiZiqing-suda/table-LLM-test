# （1）tablellama
tablellama模型性能测试。

论文宣称该模型在hitabQA数据集上实现64.71%的准确率，我们先来跑一下代码验证一下。他的eval代码运行不了，此处直接在他的inference代码里面修改，使用直接判断相等来确定是否预测正确，实际上有问题，因为模型有可能输出的内容会有多余的换行符这种，或者大小写问题。

首先使用项目原有代码复现项目。也就是T=0.6，随机种子没有设置。准确率0.5366161616161617。

之后把随机种子设置成333。T=0.6不变再跑一遍。准确率0.5378787878787878。

再使用随机种子333，T=0.01（几乎是贪心选最大的词）的复现准确率为0.5347222222222222。

使用直接严格判断相等实现了53%的准确率，再加上某些格式问题造成的误判，准确率应该确实有64%。

接下来固定随机种子为333，设置模型预测下一个token时为贪心选取最高概率的token，防止每次执行模型推理结果不同。通过筛选正确回答的样例，更换某些与问题无关的行列顺序甚至是单元格的值再次测试，看模型还能不能再次正确回答。这里的实验结果在tablellama_test文件夹里。
